Iranian Atlas Project: Design with Limited Resources

A roadmap for creating a valuable resource with efficient use of time and personnel.

Project Overview

This document outlines the design plan for the Iranian Atlas project, considering constraints on development time, AWS usage, and personnel. We'll prioritize user needs and focus on creating a valuable core experience within these limitations.

Design Documents

Here's a breakdown of key design documents that will guide development:

  • **User Stories:** These narratives describe user goals and how the Iranian Atlas will help them achieve those goals.
  • **Feature Prioritization Matrix:** This matrix will rank features based on user impact, development effort, and alignment with project goals.
  • **High-Fidelity Wireframes:** These will showcase the core functionalities and user flows of the website.
  • **Content Strategy:** This plan will define how content will be generated (text, images, maps) and how LLMs will be integrated for content creation. It will also include editorial guidelines for ensuring accuracy.

Resource Allocation

AWS Time (Several Hundred Hours):

  • **Focus:** Static website hosting on S3 for low cost and scalability.
  • **Consider:** CloudFront for CDN services (fast loading) and limited use of AWS Lambda for specific functionalities.

Developer/Researcher:

  • **Focus:** Building core functionalities, integrating LLMs for content generation.
  • **Research:** Identifying data sources for LLM training, fine-tuning prompts for accuracy.
